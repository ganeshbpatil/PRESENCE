# Module: Customer (End-Consumer Data) — **P0, minimal scope**

PRESENCE does not need a full consumer-facing customer-identity system in v1 — customer data exists only as: (a) review authorship (from GBP/Meta sync, read-only), (b) WhatsApp contact list for campaign targeting (phone number + opt-in status + basic tags). Do NOT build a consumer account/login system — that's marketplace/discovery-app territory, explicitly deferred (see BUSINESS_MODEL.md).
